script 로드 방법

<script> 로드 방법 : head tag, body tag, async, defer

<script> 태그 사용

# <head> 태그에 <script> 넣을경우

head 태그에서 스크립트가 로드될때까지 body 태그로 넘어가지 않는다.
스크립트가 로드되지 않으면 html이 화면에 그려지지 않는다.

# <body> 태그 하단에 <script> 넣을경우

html 화면이 다 그려지고 나서 스크립트를 로드드후 실행

# <script defer scr="" > 넣을경우 (비동기)

html 화면이 다 그려지 지는 동시에 스크립트를 로드한다.
html 이 다그려지면 스크립트를 실행한다.

  • - defer 은 스크립트 로딩이 완료돼도 무조건 html parsing이 끝난 후 스크립트를 실행시킨다.
  • - defer 은 먼저 작성된 스크립트가 마지막에 작성된 스크립트보다 늦게 로딩 되어도 먼저 실행시킨다. (실행 순서를 지킴)
  • - defer 은 DOM 전체가 필요한 스크립트나 실행 순서가 중요한 경우에 사용하는 것이 적합하다.

<script async scr="" > 넣을경우 (비동기)

html 화면이 그려지 지는 동시에 스크립트를 로드하고 실행한다.
스크립트가 실행될때는 html 그려지는걸 중지된다.

  • - async 는 스크립트 로딩이 완료되면 html parsing이 끝나지 않아도, 잠시 멈춰놓고 일단 스크립트를 실행시킨다.
  • - async 는 마지막으로 선언된 스크립트 파일이 제일 먼저 선언된 스크립트 파일보다 먼저 다운로드 되면 순서에 상관없이 먼저 실행되고, 이렇게 먼저 로드 된 스크립트가 먼저 실행되는 걸 'load-first order'라고 부른다.
  • - async 는 독립적으로 작동한다.
  • - async 는 방문자 수 카운터, 광고 관련 스크립트처럼 각 독립적인 역할을 하는 서드파티 스크립트를 현재 개발 중인 스크립트에 통합하려 할 때 아주 유용하다 (async 스크립트는 개발 중인 스크립트에 의존하지 않고, 그 반대도 마찬가지이기 때문)
