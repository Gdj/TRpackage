gsap.registerPlugin(ScrollTrigger)

gsap 이용하여 scrollTrigger 플러그인 적용
scrollTrigger DOC : ScrollTrigger Plugin DOC

scrollTrigger

설명
스크롤 타겟의 스크롤 위치에따라 대상 타겟에 변화를 준다.
GSAP CDN : https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/gsap.min.js
ScrollTrigger plugin CDN : https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/ScrollToPlugin.min.js
ScrollTrigger CDN : https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/ScrollTrigger.min.js
초기 선언 : gsap.registerPlugin(ScrollTrigger);
   gsap.timeline({ scrollTrigger:{ ... } }).from( ... )
gsap 옵션
gsap 속성 gsap.XXX() 속성 설명
.registerPlugin(플러그인) 플러그인 적용
.utils.toArray(".items").forEach(function($item){...} ".items"클래스를 반복적으로 각각'$item' 으로 적용.
to, from, toFrom gsap.xxx( "움직일대상 선택자", {...}, .1 ) 속성 설명
.to() .to("움직적용 선택자", { y:"300px" , ease:'none' }, 위상값(.1) )
현제 위치에서 .to({위치})로 이동
.from() .from("움직적용 선택자", { y:"300px" , ease:'none' }, 위상값(.1) )
.from({위치})에서 현제 위치로 이동
.toFrom() .toFrom("움직적용 선택자", { y:"300px" , ease:'none' }, { y:"0px" , ease:'none' } 위상값(.1) )
.toFrom(.. {위치에서}, {위치로} , ... ) 시작위치에서 끝나는 위치
to, from, toFrom - 움직일대상 선택자 : 움직임을 적용할 타겟 (".xxx", "#xxx")
- {움직임 속성값} : 움직임 속성 적용 scale:(0 ~ 0.5 ~ 1)
  • ease : easeIn, easeOut, easeInOut {none, power1~4, bounce, circ, elastic, back, expo, quad, quart, quint, sine}
- 위상값 : (기본값 : 0) `.1` 위상값이 커질수록 딜레이거리가 늘어남, 1끝나는 시점
scrollTrigger 옵션
scrollTrigger 속성 scrollTrigger 속성 설명
trigger: "" (".xxx" , "#xxx") 움직임을 트리거할 타겟 선택자
endTrigger: "" (".xxx" , "#xxx") 움직임을 끝지점 트리거할 타겟 선택자. 없으면 "trigger"
start: 'AAA BBB' AAA: 트리거 대상의 시작점, `trigger` 타겟의 크기의 위치값.
(0% == top, 50% == center, 100% == bottom)
BBB: 뷰포트 대상의 시작점, 화면 크기의 위치값.
(0% == top, 50% == center, 100% == bottom)
end: 'AAA BBB' AAA: 트리거 대상의 끝점, `trigger` 타겟의 크기의 위치값.
(0% == top, 50% == center, 100% == bottom)
BBB: 뷰포트 대상의 끝점, 화면 크기의 위치값.
(0% == top, 50% == center, 100% == bottom)
scrub: 1 - Boolean : scrub 사용 여부
- Number : "1" => 1초, 스크롤바 따라잡는 운직임의 부드러운(duration) 정도, `ease` 값대신 사용.
animation: lineTimeline ScrollTrigger에서 제어해야 하는 GSAP 트윈 또는 타임라인 인스턴스. ScrollTrigger당 하나의 애니메이션만 제어되지만, 여러 개의 ScrollTriggers가 하나의 애니메이션을 제어할 수 있습니다.
toggleClass: { 'targets': ".item", className:"active"} toggleClass : 반응에 대한 적용을 클래스로 넣기
- string : "클래스이름" - {targets : , className : ""}
toggleActions: "play none none none" (onEnter, onLeave, onEnterBack, onLeaveBack)
값("play", "pause", "resume", "reset", "restart", "complete", "reverse", and "none")사용함.
once: false 진입후 끝위치에 도달후 이벤트 삭제, 한번만 실행할지 여부. true : "play none none none"
pin: false trigger 대사을 고정할지 여부
pinSpacer: ".element" 일반적으로 ScrollTrigger는 고정된 요소를 감싸기 위해 내부적으로
를 생성하지만 고정된 요소에 iframe을 로드하는 극히 드문 시나리오에서는 ScrollTrigger가 새로 고침될 때(예: 창 크기 조정 시) iframe이 새로 고침될 수 있으므로 이 기능을 사용하면 내부적으로 생성된 요소 대신 스페이서로 사용할 요소를 지정할 수 있습니다. 이렇게 하면 ScrollTrigger가 새로 고침하는 동안 제거/추가하지 않고 iframe 콘텐츠는 그대로 유지됩니다.
markers: true 가이드 라인 표기여부 (false : default 숨김), (true : 가이드 라인표기)
onEnter, onLeave, onEnterBack, onLeaveBack
scrollTrigger Event
scrollTrigger Event scrollTrigger Event 설명
.onEnter:(()=>{...}) .onEnter: ({progress, direction, isActive})
순방향 영역에 들어올때.
.onLeave:(()=>{...}) .onLeave: ({progress, direction, isActive})
순방향 영역에 벗어날때.
.onEnterBack:(()=>{...}) onLeaveBack: ({progress, direction, isActive})
역방향 영역에 들어올때.
.onLeaveBack:(()=>{...}) onLeaveBack: ({progress, direction, isActive})
역방향 영역에 벗어날때.
.onUpdate: (self)=>{ ... } self.progress, self.direction, self.isActive
이벤트가 업데이트 되는 동안 호출됨.

		 console.clear();
 
		 const canvas = document.getElementById("hero-lightpass");
		 const context = canvas.getContext("2d");
		 canvas.width = 1158;
		 canvas.height = 770;
 
		 const frameCount = 147;
		 const currentFrame = index => (
			 `./images/${(index + 1).toString().padStart(4, '0')}.jpg`
		 );
 
		 const images = []
		 const airpods = { frame: 0 };
 
		 for (let i = 0; i < frameCount; i++) {
			 const img = new Image();
			 img.src = currentFrame(i);
			 images.push(img);
		 }
 
		 gsap.to(airpods, {
			 frame: frameCount - 1,
			 snap: "frame",
			 ease: "none",
			 scrollTrigger: {
				 scrub: 0.5
			 },
			 onUpdate: render // use animation onUpdate instead of scrollTrigger's onUpdate
		 });
 
		 images[0].onload = render;
 
		 function render() {
			 context.clearRect(0, 0, canvas.width, canvas.height);
			 context.drawImage(images[airpods.frame], 0, 0); 
		 }
 
	 </script>
